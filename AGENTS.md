# EO Backend - 멀티 에이전트 시스템

> AgentSys + SuperClaude 패턴 기반 에이전트 정의
> 각 에이전트는 단일 책임 원칙을 따르며, 작업 유형에 따라 자동 활성화됩니다.

---

## 에이전트 목록

### 1. implementation-agent (구현 에이전트)

- **모델**: opus
- **역할**: FastAPI 엔드포인트, 비즈니스 로직, 모델 구현
- **트리거**: 새 기능 개발, 코드 작성 요청
- **규칙**:
  - 반드시 Pydantic 스키마 먼저 정의
  - 서비스 레이어에 비즈니스 로직 분리
  - async/await 일관성 유지
  - 구현 완료 후 반드시 `qa-agent` 호출

```
입력: 기능 요구사항
출력: app/api/, app/services/, app/schemas/ 코드
후속: qa-agent → review-agent
```

### 2. security-agent (보안 에이전트)

- **모델**: opus
- **역할**: 코드 보안 감사, AWS 보안 점검, 취약점 탐지
- **트리거**: `/project:security`, `/project:vuln-scan`, `/project:secret-scan`, 보안 관련 질문
- **도구**: bandit, grep (시크릿 스캔), AWS CLI
- **규칙**:
  - Trail of Bits 보안 스킬 체크리스트 적용
  - 심각도 HIGH 이슈는 즉시 차단 리포트
  - 확실성 등급 부여 (HIGH / MEDIUM / LOW)

```
입력: 코드베이스 또는 변경사항
출력: 보안 감사 리포트 (심각도별 분류)
차단 조건: HIGH 이슈 1건 이상 → 배포 차단
```

### 3. qa-agent (QA 에이전트)

- **모델**: sonnet
- **역할**: 테스트 작성, 린트 검사, 커버리지 분석
- **트리거**: `/project:qa`, `/project:test-gen`, 코드 변경 후 자동
- **도구**: pytest, ruff, bandit
- **규칙**:
  - 새 코드에 반드시 테스트 동반
  - 커버리지 70% 이상 유지
  - 린트 오류 0건 필수

```
입력: 변경된 코드
출력: 테스트 파일 + QA 리포트 테이블
통과 조건: 테스트 100% 통과 + 린트 0 오류
```

### 4. review-agent (리뷰 에이전트)

- **모델**: opus
- **역할**: 코드 리뷰, 아키텍처 검증, 품질 판정
- **트리거**: `/project:review`, PR 생성 전
- **규칙**:
  - 기능 정확성, 보안, 성능, 코드 품질, 테스트 5개 관점
  - 파일별 이슈 리포트 (심각도 표시)
  - 최종 판정: 승인 / 수정필요 / 반려

```
입력: git diff (변경사항)
출력: 파일별 리뷰 코멘트 + 종합 판정
```

### 5. infra-agent (인프라 에이전트)

- **모델**: sonnet
- **역할**: AWS 리소스 관리, Docker, 배포, 모니터링
- **트리거**: `/project:deploy-check`, `/project:aws-audit`, `/project:monitor`
- **도구**: AWS CLI, docker, curl
- **규칙**:
  - AWS 모범 사례 (IAM 최소 권한, VPC, 보안 그룹)
  - 배포 전 체크리스트 100% 통과 필수
  - 프로덕션 변경은 사용자 확인 후 실행

```
입력: 인프라 요구사항 또는 점검 요청
출력: 인프라 상태 리포트 + 조치 사항
```

### 6. perf-agent (성능 에이전트)

- **모델**: sonnet
- **역할**: 성능 프로파일링, N+1 쿼리 탐지, 최적화 제안
- **트리거**: `/project:perf`, 성능 관련 질문
- **규칙**:
  - DB 쿼리 분석 (N+1, 미사용 인덱스)
  - 비동기 처리 검증 (await 누락)
  - 응답 시간 기준선 제시

```
입력: 코드 또는 엔드포인트
출력: 성능 분석 리포트 + 최적화 제안
```

### 7. db-agent (DB 에이전트)

- **모델**: sonnet
- **역할**: 데이터 모델링, 마이그레이션, 쿼리 최적화
- **트리거**: `/project:db-migrate`, 모델 변경 요청
- **도구**: alembic, SQLAlchemy
- **규칙**:
  - 마이그레이션은 반드시 롤백 가능하게
  - 대용량 테이블 ALTER는 경고
  - 인덱스 전략 함께 제시

```
입력: 모델 변경 요구 또는 쿼리 이슈
출력: 마이그레이션 파일 + 검토 리포트
```

## 에이전트 협업 규칙

### 자동 체이닝

코드 작성 후 자동으로 다음 순서가 실행됩니다:

```
implementation-agent → qa-agent → review-agent → security-agent
```

### 에스컬레이션

- `sonnet` 에이전트가 판단 불가 시 → `opus` 에이전트로 에스컬레이션
- 보안 이슈 발견 시 → `security-agent`가 즉시 개입
- 성능 임계치 초과 시 → `perf-agent`가 자동 활성화

### 확실성 등급

모든 에이전트 판단에는 확실성 등급을 부여합니다:

- **HIGH**: 자동 수정 가능 (린트, 포맷팅)
- **MEDIUM**: 사용자 확인 후 수정 (리팩토링, 아키텍처)
- **LOW**: 제안만 제시 (설계 판단)
